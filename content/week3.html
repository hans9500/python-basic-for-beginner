언제 리스트, 언제 튜플?

  • 나중에 추가/삭제할 데이터 → 리스트 (점수 모음, 장바구니)
  • 한 번 정해지고 바뀌면 안 되는 데이터 → 튜플 (좌표, RGB 색, 함수의 여러 반환값)

  • 다중 할당 a, b = ... — swap이나 함수 다중 반환에 유용.
  • 변경되어야 하면 리스트, 잠겨도 되면 튜플.
SECTION 04 · 핵심

가변 vs 불변 — 메모리 모델 깊이

3주차에서 가장 중요한 섹션

여기를 잡으면 파이썬의 객체 모델 90%가 풀립니다. 잘못 잡으면 평생 헷갈리는 버그를 만듭니다.

두 종류의 객체

파이썬의 모든 객체는 두 부류로 나뉩니다:

  • mutable (가변): 객체 자체의 내용을 바꿀 수 있음 — list, dict, set
  • immutable (불변): 한 번 만들면 절대 못 바꿈 — int, float, str, tuple, bool

핵심 차이 — append vs +

이번 주차에서 배운 모든 것을 모아 진짜 프로그램 같은 것 셋을 만들어봅니다. 공통 패턴이 등장합니다 — 데이터를 모은다(리스트/딕셔너리) → 함수로 처리한다 → 결과를 출력한다.

입력값 (미리 적기)
학생 수 + 점수들.

실습 2 — 로또 번호 생성기

1~45 중 중복 없이 6개의 숫자를 뽑습니다. random.sample()이 바로 이 일을 합니다.

lotto.py
📄 lotto.py Ctrl + Enter
실행 결과 대기 중

random.sample(range(1, 46), 6)은 1부터 45까지의 정수 중 중복 없이 6개를 뽑아 리스트로 줍니다. f"{n:2d}"는 "정수를 2자리 너비로 표시"입니다. 실행할 때마다 다른 번호가 나옵니다.

핵심 패턴 — "있으면 더하고, 없으면 1로 시작" — 가 딕셔너리 빈도 누적의 정석입니다. .split()은 문자열을 공백으로 잘라 리스트로 만듭니다 (4주차에서 자세히).

3주차 마무리

축하합니다! 3주차도 완주하셨습니다. 정리하면:

3주차에서 익힌 것
  • 리스트: [], 인덱싱, 슬라이싱, append, sort, in, for
  • 리스트의 진짜 모습: 박스 안엔 참조만, 요소는 별개 객체
  • 튜플: (), 잠긴 슬롯, 다중 할당
  • ★ 튜플의 진실: 슬롯은 잠겼지만 mutable 요소는 변할 수 있음
  • ★ 가변 vs 불변: append vs +, 같은 객체 다른 이름표
  • 딕셔너리: 키-값 매핑, 빈도 누적 패턴
  • 함수 def: 매개변수, return, 다중 반환
  • 매개변수의 모든 것: 위치/키워드/기본값/다중 할당

4주차에서는 None, 리스트·문자열 심화 도구, 그리고 외부 인터넷에서 데이터를 가져오는 API 호출로 가볍게 마무리합니다.
