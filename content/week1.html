Week 1 · Lesson 1

컴퓨터, 객체, 그리고 첫 코드

파이썬 코드를 한 줄도 쓰기 전에, AI 시대에 왜 코딩 기초가 필요한지부터 솔직하게 이야기합니다. 그리고 코드가 실행될 때 컴퓨터 안에서 무슨 일이 일어나는지, 변수는 도대체 무엇인지 천천히 살펴봅니다.

SECTION 00

AI 시대에 파이썬을 왜?

Claude, ChatGPT, Gemini 같은 AI가 코드를 척척 만들어주는 시대입니다. "그러면 코딩은 더 이상 안 배워도 되는 거 아니야?" 하는 생각이 자연스럽게 들죠. 네 가지만 짚어보겠습니다.

1. AI에게 정확히 시키려면 알아들을 줄 알아야 합니다

회사 직원에게 "더 빠르게 처리해주세요"라고만 하면 그 직원도 어디서 어떻게 빠르게 할지 모릅니다. "사진 로딩이 3초 걸리는데 1초로 줄여달라"고 해야 일이 진행되죠.

AI도 똑같습니다. "엑셀 자동화해줘"라고만 하면 흐릿한 답이 나옵니다. "이 폴더의 모든 .xlsx 파일에서 B열의 합계를 구해서 새 파일에 정리해줘"처럼 구체적으로 말해야 정확한 코드가 나옵니다. 이런 지시를 만들려면 "파일이 뭔지", "열이 뭔지", "합계 같은 처리 단위가 뭔지"를 어느 정도는 알아야 합니다.

2. AI는 자신감 있게 틀린 답을 줍니다

AI는 "이 코드는 정확합니다"라고 확신하며 틀린 답을 종종 줍니다. 코드가 안 돌아가는데도 "맞다"고 우기기도 합니다.

이걸 검증할 사람은 결국 본인입니다. 기본 문법을 알면 코드를 한 번 훑어보고 "어, 이건 좀 이상한데?"를 알아챌 수 있습니다. 모르면 그냥 받아서 쓰다가 데이터가 망가지는 일도 생깁니다.

3. AI가 못 하는 일이 의외로 많습니다

AI는 사용자 컴퓨터에 직접 접근할 수 없습니다. "내 PC의 이 파일을 열어서 분석해줘"를 AI 혼자서는 못 합니다. 결국 AI가 만들어준 코드를 본인이 실행해야 합니다.

코드 한 줄을 어디에 어떻게 붙여넣고 실행하는지 — 이게 의외로 진입 장벽입니다. 이번 4주 강의의 목표가 그 진입 장벽을 함께 넘는 것입니다.

4. 자동화의 가치 — 한 번 짜두면 평생 일합니다

매주 같은 보고서 양식을 만드시나요? 매월 같은 정리를 반복하시나요? 이런 반복 업무는 코드 한 번 짜두면 버튼 한 번에 자동으로 처리됩니다. 엑셀 매크로보다 강력하고, 그 코드는 AI가 거의 다 만들어줍니다. 단, 본인이 실행할 줄은 알아야 합니다.

한 줄 요약

AI 시대에는 "AI에게 일을 시키는 사람"과 "AI가 일을 시키는 사람"이 나뉩니다. 코딩 기초를 아는 쪽이 전자입니다. 깊이 배우자는 게 아니라, AI와 협력할 수 있을 만큼만 배우자는 게 이 강의의 목표입니다.

SECTION 01

컴퓨터 안 들여다보기

노트북, 데스크톱, 스마트폰 — 모양은 달라도 안에서 일하는 방식은 비슷합니다. 컴퓨터 안에는 GPU, 메인보드, 전원 공급장치 등 여러 부품이 있지만, 파이썬 코드가 실행될 때 가장 자주 등장하는 세 곳이 있습니다. 이번 강의 동안 계속 만나게 될 곳들이니, 먼저 익혀두고 가겠습니다.

CPU · 메모리 · 저장장치

CPU (중앙처리장치)

실제로 계산과 판단을 수행하는 부품입니다. 덧셈·뺄셈 같은 산술 연산, 두 값이 같은지 비교하기, 다음에 어떤 명령을 실행할지 정하기 같은 일을 합니다. 매우 빠르지만 — 1초에 수십억 번 명령을 처리합니다 — 자체적으로 데이터를 오래 보관하는 공간은 거의 없습니다. 값을 다루려면 메모리에서 읽어와야 하고, 결과를 보존하려면 다시 메모리에 써야 합니다.

메모리 (RAM, Random Access Memory)

지금 실행 중인 프로그램과 그 데이터가 머무는 곳입니다. CPU가 즉시 접근 가능한 작업 공간이라고 생각하면 됩니다. 빠르지만 용량이 비교적 작고(보통 8~64GB), 전원이 꺼지면 내용이 사라집니다.

저장장치 (SSD/HDD)

모든 파일이 영구적으로 보관되는 곳입니다. 사진, 문서, 설치된 프로그램, 운영체제 — 전부 여기 있습니다. 메모리보다 용량이 훨씬 크지만(수백 GB ~ 수 TB), 읽고 쓰는 속도는 메모리보다 느립니다. 전원이 꺼져도 내용이 보존됩니다.

왜 이렇게 나뉘어 있을까?

"전부 빠르고 큰 거 하나로 만들면 안 돼?"라고 묻고 싶을 겁니다. 빠른 부품일수록 비싸고 용량이 작기 때문입니다. 그래서 역할을 나눠 협력하게 만드는 게 가장 효율적입니다 — 이걸 메모리 계층(memory hierarchy)이라고 부릅니다.

그림으로 보기

그림 1.1 — CPU·메모리·저장장치의 역할과 데이터 흐름

그림에서 핵심은 화살표의 방향입니다. CPU는 메모리하고만 직접 대화합니다. 저장장치의 파일을 다루고 싶으면 먼저 메모리로 불러와야 CPU가 작업할 수 있습니다. "사진 열기"가 한순간이 아니라 잠깐 로딩이 걸리는 이유가 이것입니다 — 저장장치에서 메모리로 사진 데이터를 옮기는 시간이죠.

이 섹션의 핵심
  • CPU는 계산과 판단을 하는 부품. 빠르지만 데이터는 거의 보관 못 함.
  • 메모리(RAM)는 지금 실행 중인 데이터가 머무는 작업 공간. 전원 끄면 사라짐.
  • 저장장치(SSD/HDD)는 파일이 영구 보관되는 큰 창고. 쓰려면 일단 메모리로 가져와야 함.
SECTION 02

프로그램이 실행된다는 것

"더블클릭으로 프로그램을 실행한다" — 자주 쓰는 표현이지만, 실제로는 컴퓨터 안에서 여러 단계가 일어납니다. 천천히 따라가 보겠습니다.

3단계로 보는 프로그램 실행

그림 1.2 — 프로그램 실행의 3단계

1단계 — 저장장치에서 메모리로 적재

프로그램은 평소에는 저장장치(SSD/HDD)에 파일로 잠자고 있습니다. 실행 버튼을 누르면 운영체제가 그 파일을 메모리로 복사합니다. 이게 "프로그램이 실행 중이다"의 시작입니다. 메모리에 올라가 있어야 CPU가 손댈 수 있으니까요.

2단계 — CPU가 명령을 한 줄씩 실행

메모리에 올라간 프로그램은 결국 "명령어들의 목록"입니다. CPU는 그 목록의 첫 번째 명령부터 차례로 읽으며 실행합니다. "메모리의 100번 자리 값과 200번 자리 값을 더해라", "그 결과가 0이면 300번 명령으로 점프해라" 같은 단순한 명령들의 연속입니다.

3단계 — 결과를 메모리에 쓰거나 화면으로 출력

명령을 실행한 결과는 다시 메모리에 저장되거나, 화면·스피커 같은 출력 장치로 흘러갑니다. 우리가 화면에서 보는 모든 것은 이 과정의 산물입니다.

파이썬은 좀 다릅니다

위 설명은 일반적인 프로그램(예: 컴파일된 C 프로그램) 기준입니다. 파이썬은 한 단계가 더 있습니다 — 파이썬 인터프리터(Python interpreter)가 먼저 실행되고, 그 인터프리터가 우리가 작성한 .py 파일을 한 줄씩 통역합니다. 그래서 같은 코드도 컴퓨터마다 똑같이 동작합니다.

SECTION 03

파이썬과 객체의 세계

이제부터가 진짜 파이썬 이야기입니다. 그리고 가장 중요한 개념입니다. 여기를 잘 잡고 가야 변수, 함수, 리스트, 딕셔너리 — 앞으로 나올 모든 것이 자연스럽게 이해됩니다.

한 문장으로

파이썬에서는 모든 값이 "객체(object)"입니다.

그 전에 — 메모리 안은 어떻게 생겼나?

s1에서 메모리(RAM)는 "지금 실행 중인 프로그램과 그 데이터가 머무는 곳"이라고 했죠. 그 메모리 안에 들어가 보면, OS는 한 프로그램에게 메모리 공간을 주면서 역할별로 영역을 나눠둡니다.

그림 1.3 — 한 프로그램이 받은 메모리 공간의 구조

각 영역의 역할을 짧게 정리하면:

  • TEXT (코드 영역) — 실행할 명령어들이 들어있는 곳. print, len 같은 함수의 코드도 여기에 있습니다.
  • GLOBAL/STATIC (전역 영역) — 프로그램 전체에서 쓰이는 데이터. 모듈 수준에서 만든 변수들의 흔적이 여기 있습니다.
  • HEAP — 모든 객체가 만들어지는 곳입니다. 우리가 5라고 쓰는 순간 만들어진 정수 객체, "안녕"이라고 쓴 문자열 객체 — 전부 HEAP에 산다고 보시면 됩니다.
  • STACK — 함수가 호출될 때 쓰는 작업 공간. 함수 안에서 만든 변수의 이름표(레퍼런스)들이 여기 일시적으로 머뭅니다.
참고

이 그림은 일반적인 메모리 구조입니다. 파이썬(CPython) 내부 구현은 더 복잡하지만, "객체는 HEAP에 산다"는 사실은 그대로입니다. 다음 섹션부터 자주 "객체"라는 말이 나오면, "아, HEAP에 만들어진 그것"이라고 생각하시면 됩니다.

객체란 무엇인가?

파이썬에서 객체(object)란 메모리(HEAP)에 만들어진 데이터 덩어리입니다. 숫자 1도 객체이고, 글자 "안녕"도 객체이고, 참/거짓 True도 객체이고, 함수 print도 객체입니다. 파이썬에서 우리가 다루는 모든 값은 HEAP에 객체로 만들어져 있습니다.

각 객체는 다음 세 가지를 가지고 있습니다:

그림 1.4 — 모든 객체는 타입, 값, 고유 번호(id)를 가짐

① 타입 (type)

객체가 어떤 종류의 데이터인지를 나타냅니다. int(정수), str(문자열), float(실수), bool(참/거짓), list(리스트) 등이 있습니다. 타입은 그 객체가 어떤 일을 할 수 있는지를 정합니다 — 예를 들어 정수는 더하기/빼기를 할 수 있고, 문자열은 이어붙이기를 할 수 있죠.

② 값 (value)

객체가 실제로 가지고 있는 데이터입니다. 정수 객체의 값은 숫자 5, 문자열 객체의 값은 글자 "안녕" 등이죠.

③ 고유 번호 (id)

객체가 메모리 어디에 자리 잡고 있는지를 나타내는 번호입니다. 같은 시점에 같은 번호를 가진 객체는 단 하나뿐입니다 — 주민등록번호 같은 거라고 생각하면 됩니다. 이 번호는 파이썬이 알아서 정해주고, id()라는 함수로 확인할 수 있습니다.

실행하면 <class 'int'>, <class 'str'>, <class 'float'> 같은 게 나옵니다. 이건 "이 객체의 타입은 int 클래스다"라는 뜻입니다 — 클래스(class)는 타입의 다른 표현이라고 보시면 됩니다.

그리고 id(5)가 보여주는 큰 숫자는 그 객체의 메모리 주소입니다. 여러분이 실행할 때마다 다른 숫자가 나올 수 있고, 같은 객체라면 같은 번호가 나옵니다.

왜 이게 중요한가?

객체의 정체성(타입·값·id)을 이해하면, 다음 섹션에서 다룰 "변수란 무엇인가"가 자연스럽게 풀립니다. 파이썬의 변수는 객체를 "담는 상자"가 아닙니다 — 그 진짜 정체를 곧 만나게 됩니다.

이 섹션의 핵심
  • 파이썬에서 모든 값은 객체다 — 숫자, 문자, 함수 모두.
  • 각 객체는 타입(type), 값(value), 고유 번호(id)를 가진다.
  • type()으로 타입을, id()로 고유 번호를 확인할 수 있다.

줄바꿈에 대해

print()는 매번 호출할 때마다 자동으로 줄을 바꿉니다. 그래서 두 번 호출하면 두 줄로 출력됩니다. 이걸 바꾸고 싶으면 end 매개변수를 쓸 수 있는데, 지금은 "기본은 줄바꿈"이라는 사실만 기억하시면 됩니다.

print()도 객체입니다

아까 "모든 게 객체다"라고 했죠. print도 객체 — 구체적으로는 함수 객체(function object)입니다. 뒤에 붙은 괄호 ()는 "이 함수 객체를 호출해라"는 의미입니다.

함수 호출의 사실

print("안녕")은 두 단계입니다 — ① print라는 함수 객체를 메모리에서 찾고, ② 거기에 "안녕"이라는 문자열 객체를 넘겨줘서 호출합니다. 함수 호출은 늘 함수 이름 + 괄호 안에 객체 전달의 패턴입니다.

이 섹션의 핵심
  • print(객체)는 객체의 값을 화면에 표시한다.
  • 여러 값은 쉼표로 구분 — 공백 한 칸으로 자동 이어붙여 출력.
  • print()는 매 호출마다 자동으로 줄바꿈한다.
  • print 자체도 함수 객체. 괄호는 호출의 표시.
SECTION 05 · 핵심

변수 = 객체를 가리키는 이름표

이번 섹션이 1주차에서 가장 중요합니다

파이썬의 변수는 "값을 담는 상자"가 아닙니다. 이걸 처음에 잘 잡아두면 앞으로 헷갈릴 일이 없고, 잘못 잡으면 평생 헷갈립니다.

변수란?

프로그래밍에서 변수(variable)는 어떤 값을 다시 사용하기 위해 이름을 붙여둔 것입니다. 한 번 만들어둔 객체를 다시 찾으려면 어떻게든 참조할 방법이 필요합니다 — 그게 변수입니다.

그런데 "어떻게 참조하는가"가 언어마다 다릅니다.

C 언어의 변수 (참고용)

C 언어에서는 변수가 값을 담는 상자처럼 동작합니다. int x = 5;라고 쓰면 메모리에 "x"라는 이름의 네모난 공간이 생기고, 그 안에 5가 직접 들어갑니다.

그림 1.5 — C 언어 변수: 상자에 값을 직접 담음

파이썬의 변수 — 이름표 모델

파이썬은 다릅니다. 파이썬에서 x = 5를 쓰면 두 가지 일이 일어납니다:

  1. 메모리에 정수 객체 5가 만들어짐
  2. x라는 이름표가 만들어져 그 객체를 가리킴

값은 객체에 들어있고, 변수는 그 객체를 가리키는 이름표일 뿐입니다.

그림 1.6 — 파이썬 변수: 객체를 가리키는 이름표

왜 이렇게 다를까?

이름표 모델 덕분에 파이썬은 다음과 같은 일이 자연스럽습니다:

  • 같은 객체에 여러 이름표 붙이기 (y = x)
  • 이름표를 다른 객체로 옮기기 (x = 10)
  • 이름표 없는 외로운 객체는 자동으로 메모리에서 사라짐 (가비지 컬렉션)

비유: 사물함과 이름표

이걸 쉽게 떠올리려면 사물함과 이름표를 생각하세요. 객체(메모리에 저장된 값)는 사물함입니다. 변수는 그 사물함에 붙인 이름표죠.

  • 사물함(객체)은 메모리에 자리 잡고 있습니다.
  • 이름표(변수)는 어떤 사물함을 가리킵니다.
  • 같은 사물함에 이름표 두 개 붙일 수 있습니다.
  • 이름표를 다른 사물함으로 옮길 수도 있습니다.
  • 아무 이름표도 안 붙은 사물함은 비워져 사라집니다.

변수 이름 짓기

변수 이름에는 규칙이 있습니다:

  • 알파벳, 숫자, 밑줄(_)만 사용 가능
  • 숫자로 시작할 수 없음 (2name 안 됨)
  • 대소문자 구분 (name과 Name은 다른 변수)
  • 한글도 가능하지만 일반적으로 영문 권장
  • 의미가 분명한 이름이 좋음 (a보다 age)
이 섹션의 핵심 (가장 중요!)
  • 파이썬 변수는 값을 담는 상자가 아니라 객체를 가리키는 이름표입니다.
  • x = 5 = 정수 객체 5를 만들고, x 이름표가 그것을 가리키게 한다.
  • y = x = 같은 객체에 y라는 이름표 하나 더 붙인다 (id 같음).
  • x = 10 = x 이름표를 새 객체로 옮긴다 (기존 객체는 그대로).
