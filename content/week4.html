지금까지 변수에는 항상 값이 있었습니다. 그런데 가끔 "아직 값이 없음"을 표현해야 할 때가 있습니다. 파이썬에는 그런 상태를 위한 특별한 객체가 있습니다 — None.

JSON이란?

JSON(JavaScript Object Notation)은 데이터를 문자열로 표현하는 형식입니다. 인터넷에서 데이터를 주고받을 때 가장 흔히 쓰입니다. 모양이 파이썬 딕셔너리/리스트와 매우 비슷합니다.

핵심 함수는 두 개:

  • json.loads(문자열) — JSON 문자열을 파이썬 딕셔너리/리스트로
  • json.dumps(객체) — 그 반대 (파이썬 객체를 JSON 문자열로)

이름이 헷갈릴 수 있는데 "loads = load string", "dumps = dump string"으로 외우시면 됩니다.

각각을 한 줄로만 짚으면:

  • 클래스 / OOP — "데이터와 그 데이터를 다루는 동작을 한 묶음으로". 큰 프로그램에서 필수.
  • 데코레이터 — 함수 위에 @something으로 기능 덧붙이기. 웹 프레임워크에서 자주 등장.
  • 람다 — 이름 없는 한 줄 함수. lambda x: x*2. 4주차 s4의 multi_city.py에서 살짝 봤어요.
  • 컴프리헨션 (깊이) — 3주차 로또에서 살짝 본 그것. if 필터, 중첩, 딕셔너리 컴프리헨션까지.
  • 제너레이터 / yield — 값을 한꺼번에 만들지 않고 필요할 때마다 하나씩.
  • 비동기 / async ★ — 4주차에서 만난 await가 이쪽의 입구. async def, 동시 처리 등이 본격 등장.
  • 타입 힌트 — def f(x: int) -> str:. 코드의 의도를 명시. 큰 프로젝트에서 가독성 큰 도움.
  • try/except 심화 — 에러 종류별로 다르게 처리. 4주차 s3에서 잠깐 본 try/except의 깊이.
  • 파일 I/O — 디스크의 파일을 읽고 쓰기. 실무에서 매우 흔함.
  • 모듈 / 패키지 — 내 코드를 여러 파일로 나눠 관리. import의 깊이.
  • 정규식 — 문자열 패턴 검색의 끝판. 4주차 s3 이메일 검증을 진짜 정밀하게 하려면 이게 필요.
  • 가상환경 / pip — 프로젝트마다 분리된 라이브러리 관리.
