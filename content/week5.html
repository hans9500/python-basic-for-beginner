Week 5 · Lesson 1 · 부록

실무 IT 산업의 큰 그림

4주 동안 Python을 손에 익혔으니, 이제 IT 산업의 큰 그림을 한번 둘러봅시다. 그림 한 장에 모든 것이 들어 있어요. "이걸 다 알아야 하나?" 부담 없이, "이런 세계가 있구나" 정도만 보시면 됩니다.

SECTION 01

왜 큰 그림을 봐야 할까요?

여러분이 매일 쓰는 인스타, 카톡, 배달의민족, 유튜브 — 이 서비스들은 Python 한 가지 언어로만 만들어지지 않습니다.

수십 개의 언어, 수백 개의 도구가 함께 움직여서 "주문하기" 버튼 한 번이 작동해요. 이걸 한 사람이 다 다루지는 않습니다. 백엔드·프론트엔드·DevOps·데이터 엔지니어가 각자 한두 영역에 집중하고 협력합니다.

그래서 왜 큰 그림을 봐야 하느냐

"백엔드", "DB", "API", "캐시" 같은 단어를 들었을 때 어디 자리에 있는 도구인지 안다는 것이 큰 차이를 만듭니다.

AI가 "Redis 써서 캐시 처리하시면 됩니다"라고 답했을 때, "왜 캐시가 필요한지" 알면 그 답을 검증할 수 있어요.

SECTION 02

한 장으로 보는 IT 산업

인터넷 서비스는 보통 3개 영역이 협력해서 만들어집니다.

  • Frontend — 사용자가 보는 화면
  • Backend — 보이지 않는 처리
  • Database / 뒷단 서비스 — 데이터 보관 + 주변 서비스

아래 한 장에 모든 것이 들어 있어요.

그림 5.1 — IT 산업 한 장. Frontend(왼쪽) ↔ Backend(가운데) ↔ Database/뒷단(오른쪽)

Web 기초 3종 세트

Web을 배우려면 HTML, CSS, JavaScript 셋이 한 묶음입니다.

  • HTML — 뼈대. 어떤 글·이미지·버튼이 있는지.
  • CSS — 옷. 색깔·크기·위치 같은 모양.
  • JavaScript — 움직임. 버튼 누르면 반응하는 등.

건물에 비유하면 골조 / 외관 / 전기·엘리베이터입니다.

큰 사이트는 프레임워크를 씁니다

HTML/CSS/JS만으로는 큰 사이트가 복잡해져요. 그래서 이미 만들어진 도구 모음(프레임워크)을 사용합니다.

  • React — 페이스북이 만듦. 가장 인기. 인스타 웹·넷플릭스·에어비앤비.
  • Vue — 배우기 쉬움. 한국에서도 인기.
  • Angular — 구글. 대형 엔터프라이즈.
  • Svelte — 새로 부상. 가벼운 결과물.

모바일 앱 — iOS와 Android

각 OS마다 전용 언어가 있습니다.

  • iOS (아이폰) — Swift. 애플이 만든 언어.
  • Android (삼성·LG 등) — Kotlin. 구글의 공식 언어.

같은 앱을 두 번 만들기 부담스러우면 크로스 플랫폼 도구를 씁니다.

  • React Native — JavaScript로. 페이스북.
  • Flutter — Dart 언어로. 구글.

데스크톱 앱 — Electron

PC 프로그램은 요즘 Electron(웹 기술로 데스크톱 앱)으로 많이 만듭니다.

VS Code, Slack, Discord, Figma 데스크톱이 모두 Electron이에요.

SECTION 04 · Backend

Backend — Python이 사는 곳

인스타에서 좋아요 버튼을 누르면, 그 정보가 인스타 회사의 서버 컴퓨터까지 갑니다.

그 서버에서 데이터베이스에 저장되고, 작성자에게 알림이 보내지고, 피드 정렬이 다시 계산됩니다. 이 모든 처리가 Backend의 일이에요.

"서버"라는 단어 두 가지 의미

  • 서버 컴퓨터 — 회사가 24시간 켜놓고 인터넷에 연결한 큰 컴퓨터.
  • 서버 소프트웨어 — 그 컴퓨터에서 도는 백엔드 프로그램.

보통 둘 다를 묶어서 "서버"라 부릅니다.

Backend의 5가지 일

  1. 요청 받기 (Frontend의 API 요청을 받음)
  2. 인증 확인 (진짜 본인인가? 토큰 검증)
  3. 비즈니스 로직 처리 (할인 쿠폰 적용 같은 규칙)
  4. DB 입출력 (데이터 저장·조회)
  5. 응답 반환 (결과를 JSON으로 돌려줌)

Backend 언어들

Backend는 다양한 언어로 만들 수 있어요.

  • Python ★ — 여러분이 배운 그것! Django/Flask/FastAPI. 데이터·AI 연계 강점. Instagram·Netflix 등에서 사용합니다.
  • Java — 한국 기업 표준. Spring Boot. 네이버·카카오·토스·배민 모두 사용.
  • JavaScript/TypeScript — Node.js. Frontend와 같은 언어로 백엔드.
  • Go — 구글이 만듦. 동시 처리 강함.
  • Ruby — Ruby on Rails. GitHub·Airbnb·Shopify가 시작.
  • PHP — Facebook·Wikipedia가 시작한 옛 언어. 여전히 큰 비중.
  • C# — 마이크로소프트의 .NET.
  • Rust — 새로 부상. 안전성·성능 강함.

Python으로 시작하는 백엔드

여러분은 4주 동안 Python을 배웠으니, 다음 도구 중 하나로 백엔드 개발자로 시작할 수 있어요.

  • Flask — 가장 작고 학습용으로 좋음.
  • FastAPI — 현대적이고 빠름. 최근 가장 인기.
  • Django — 큰 사이트용. Instagram도 사용.

큰 백엔드는 마이크로서비스로

배민 같은 큰 서비스는 백엔드를 한 덩어리로 만들지 않습니다. 작은 조각들로 나눠서 운영해요 — 이걸 마이크로서비스라고 합니다.

예: 회원 서비스 / 가게 서비스 / 주문 서비스 / 결제 서비스 / 배달 서비스 / 알림 서비스. 한 서비스가 잠깐 멈춰도 다른 서비스에 영향이 없어요.

배달의민족은 마이크로서비스 + Java/Spring 조합으로 운영됩니다 (출처: 우아한형제들 기술블로그 2016).

SECTION 05 · Database / 뒷단

Database / 뒷단 서비스 — 백엔드 뒤의 세계

백엔드는 혼자 일하지 않습니다. 뒤에는 데이터 보관·캐시·메시지 전달·결제·AI 등이 줄지어 있어요.

Database

회원 정보, 게시글, 주문 내역… 사용자가 입력한 모든 데이터가 DB에 영구 저장됩니다.

크게 두 종류로 나뉘어요.

관계형 DB (RDB)

데이터를 표(테이블)로 저장합니다. 표끼리 서로 연결지을 수 있어서 "관계형"입니다.

배달앱이라면 users 표 / stores 표 / orders 표 같은 식. orders가 user_id로 users를 가리키는 구조예요.

은행·쇼핑몰·예약 시스템처럼 데이터가 정확하게 맞아야 하는 곳에 강합니다.

  • MySQL — 가장 흔함. 배달의민족도 AWS Aurora MySQL 사용 (출처: 우아한형제들 기술블로그 2019).
  • PostgreSQL — 기능 풍부. 새 프로젝트가 많이 선택.
  • Oracle — 유료. 대형 은행·금융권.

NoSQL

유연한 구조가 강점입니다.

  • MongoDB — JSON 형태. 상품마다 속성이 다른 쇼핑몰에 적합.
  • Cassandra — 여러 서버에 분산. 인스타·Netflix처럼 대용량·급증 데이터.
  • DynamoDB — AWS의 관리형 DB.
SQL — 백엔드의 다음 학습

관계형 DB에 접근하려면 SQL이라는 언어를 씁니다. 예: SELECT * FROM users WHERE age >= 30; (30살 이상 다 찾기).

모든 RDB 제품(MySQL·PostgreSQL·Oracle)이 SQL을 써요. 백엔드 개발자가 다음으로 배워야 할 가장 중요한 한 가지가 SQL입니다.

캐시

캐시(Cache)는 자주 쓰는 데이터를 미리 빠른 곳에 저장해두는 방법입니다.

인스타 피드를 매번 DB에서 처음부터 계산하면 너무 느려요. 한 번 계산한 피드를 메모리에 잠깐 저장해두면 다음에 0.01초 만에 꺼낼 수 있습니다.

표준 도구가 Redis예요.

Redis가 표준이 된 이유

한 도구로 여러 일을 할 수 있어서 다들 Redis 하나는 둡니다.

  • 자주 쓰는 데이터 빨리 꺼내기 (캐시 — 가장 흔한 용도)
  • 로그인 세션 보관
  • 실시간 순위 (게임 점수판, 인기 상품)
  • 작은 메시지 큐 흉내
  • 여러 서버 간 동시 접근 제어 (분산 락)

메시지 큐

한 시스템에서 다른 시스템으로 메시지 보낼 때 줄 세워주는 우편함입니다.

왜 필요한가?

배민에 주문이 들어오면 백엔드가 매장·라이더·푸시·분석 4곳에 알려야 해요.

직접 차례로 호출하면 한 곳이라도 멈추면 다 멈추고 응답이 늦어집니다.

메시지 큐에 "주문 들어왔어!" 한 번만 던지면 4곳이 각자 알아서 가져갑니다. 한쪽이 멈춰도 다른 쪽 영향 없어요.

표준 도구는 Kafka입니다. 큰 규모 처리에 강해요.

LinkedIn·Netflix·Uber·배달의민족이 모두 사용합니다 (출처: 우아한형제들 기술블로그 2025.6).

작은 규모는 RabbitMQ를 더 자주 씁니다 — 가볍고 셋업이 간단해서요.

검색 엔진

DB에도 검색 기능이 있지만 텍스트 검색은 느립니다.

예: 배민에서 "치킨" 검색 시 가게명·메뉴명·비슷한 단어·오타 보정·인기순 정렬이 동시에 필요해요. 이걸 일반 DB가 처리하면 매우 느립니다.

표준 도구는 Elasticsearch. 쿠팡·GitHub·배민 가게 검색 등에 쓰입니다.

결제 — PG사

PG사(Payment Gateway, 결제 대행사)는 온라인 결제를 대신 처리하는 전문 회사입니다.

한국법상 쇼핑몰이 카드 정보를 직접 가지면 불법이라, 전문 회사에 맡기는 구조예요.

흐름: 쇼핑몰 → PG사 → 카드사 → 은행 → PG사 → 쇼핑몰.

한국에는 NHN KCP, KG이니시스, 토스페이먼츠, 나이스페이먼츠 같은 회사가 있어요.

그 외에 간편결제로 카카오페이·네이버페이·페이코도 있습니다.

푸시 알림

앱이 닫혀있어도 화면 위에 뜨는 메시지 (카톡 알림 같은 것).

푸시는 직접 못 보내고 운영체제 회사를 거쳐야 합니다.

  • Android — 구글의 FCM (Firebase Cloud Messaging). 무료.
  • iOS — 애플의 APNs (Apple Push Notification service). 무료.

클라우드

옛날엔 회사가 서버 컴퓨터를 직접 사서 운영했어요. 지금은 클라우드에서 빌려씁니다.

"내가 직접 안 사고, 인터넷 너머의 회사가 운영하는 컴퓨터를 빌려쓰는" 방식. 사용한 만큼만 비용을 내요.

빅3가 시장을 차지합니다.

  • AWS (아마존) — 가장 오래되고 서비스 많음. 한국 IT 기업 대부분 사용 (배민·토스·당근·쿠팡).
  • Azure (마이크로소프트) — 기업 시장 강함.
  • GCP (구글) — AI·머신러닝 강함.

한국에는 네이버 클라우드, NHN Cloud도 있어요. 공공기관·금융권이 선호합니다.

컨테이너

"내 컴에선 잘 돌았는데요"

개발자 노트북에선 코드가 잘 돌았는데 회사 서버에선 안 도는 경우가 많았어요. Python 버전·라이브러리·환경이 달라서요.

Docker는 코드+필요한 환경을 통째로 박스(컨테이너)로 묶어서, 어떤 컴퓨터에 가져가도 똑같이 돌게 해줍니다. "내 컴에선 잘 돌았는데" 문제가 사라졌어요.

Kubernetes(쿠버네티스, k8s)는 컨테이너 수십·수백 개를 자동 관리. 한 컨테이너 죽으면 자동 재시작, 트래픽 몰리면 자동 확장.

★ AI 서비스

IT 산업에 새로운 카테고리가 생겼습니다 — AI 서비스예요.

백엔드 개발자가 외부 AI를 호출해서 자기 서비스에 똑똑함을 더하는 게 새 표준이 됐습니다.

  • Anthropic Claude — 코드 생성에 강함. 기업 시장에서 빠르게 성장.
  • OpenAI ChatGPT — 일반인 인지도 1위.
  • Google Gemini — 멀티모달(텍스트·이미지·동영상) 강함.
  • Meta Llama — 오픈소스. 무료 다운로드해서 자기 서버에 깔 수 있음.

여러분이 배운 Python으로 Claude나 OpenAI API를 호출하는 게 지금 가장 인기 있는 신입 프로젝트 중 하나예요.

4주차에서 배운 pyfetch가 그대로 적용됩니다.

SECTION 06

시뮬레이터 — 배달의민족 주문 흐름

위에서 본 모든 게 어떻게 함께 움직이는지 시뮬레이터로 따라가 봅시다.

배달의민족 주문 한 번에 일어나는 일을 10단계로 보여드려요.

  • ✓ 표시 — 우아한형제들이 공개한 사실 (출처 명시)
  • ❓ 표시 — 공개되지 않아 일반 IT 패턴으로 추측한 부분

주문 한 번에 8개 시스템 협력

모바일 앱 + 백엔드 + DB + 결제 + 메시지 큐 + 매장 + 라이더 + 푸시 — 8개 시스템이 협력합니다.

이건 배민 한 회사 안의 시스템들만 셈한 것. 클라우드(AWS)·결제 카드사·푸시 시스템(FCM/APNs)까지 더하면 더 많아요.

인스타 사진 업로드, 카톡 메시지, 유튜브 영상 — 다른 서비스도 같은 다단계 흐름 위에서 동작합니다.

"내가 다 알아야 하나?" — 아닙니다

한 사람이 다 다루지 않습니다. 백엔드·프론트엔드·DevOps·데이터·모바일 개발자가 각자 한두 영역에 집중하고 협력해요.

여러분이 Python으로 시작했다면 Backend 또는 뒷단 일부에 자연스럽게 닿게 됩니다.

SECTION 07

여기까지 — 그리고 앞으로

부록까지 따라오시느라 수고 많으셨습니다.

이제 여러분은 단순히 Python 문법을 아는 사람이 아니라, "인터넷 서비스가 어떻게 돌아가는지 큰 그림을 본 사람"입니다.

1주차의 약속 회수 — AI 시대

1주차에 약속드렸죠. "AI 시대에는 'AI에게 일 시키는 사람'과 'AI가 일 시키는 사람'이 나뉜다"고요.

  • AI가 만들어준 코드가 어디에 들어가는지 알아야 검증할 수 있어요.
  • "백엔드", "DB", "API", "캐시" 같은 단어를 들으면 어느 자리인지 안다는 게 큰 차이예요.
  • AI가 "Redis 써서 캐시 처리하시면 됩니다"라고 답해도, 여러분은 왜 캐시가 필요한지 알 수 있죠.
  • AI 서비스(Claude·ChatGPT·Gemini) 자체도 결국 백엔드가 호출하는 뒷단 서비스예요.

다음에 갈 수 있는 길

  • Python 더 깊이 — 클래스, 데코레이터, 람다, 컴프리헨션, 비동기.
  • 백엔드 시작 — Flask 또는 FastAPI로 작은 API 서버 만들기.
  • SQL과 데이터베이스 — PostgreSQL이나 MySQL로 직접 표 만들고 쿼리.
  • Frontend — HTML/CSS/JS 기초 → React.
  • AI 활용 — Claude나 OpenAI API를 Python으로 호출하는 작은 프로젝트.
  • 클라우드 — AWS 무료 계정으로 작은 서버 띄우고 코드 돌려보기.

실무 IT 산업에는 끝없이 새로운 도구가 나옵니다. 5년 후엔 지금 본 도구들 중 일부는 사라지고 새 도구가 자리잡을 거예요.

하지만 Frontend ↔ Backend ↔ Database라는 큰 그림은 변하지 않습니다. 이름만 바뀔 뿐 같은 자리에 같은 종류의 도구가 자리잡아요.

여러분이 4주 동안 배운 변수·함수·반복·조건·자료구조도 마찬가지입니다. 어떤 언어를 써도, 어떤 시대가 와도, 이 기본은 변하지 않아요.

수고 많으셨습니다. 다음 단계로 가는 길에서 또 만나요.

5주차 핵심 정리
  • 인터넷 서비스의 큰 그림: Frontend ↔ Backend ↔ Database/뒷단.
  • Frontend: HTML/CSS/JS, React, Vue, Swift, Kotlin, Flutter.
  • Backend: Python, Java, JavaScript, Go, Ruby, PHP, C#, Rust. 큰 백엔드는 마이크로서비스로.
  • Database: MySQL, PostgreSQL, MongoDB, Oracle.
  • 뒷단: 캐시(Redis), 메시지 큐(Kafka), 검색(Elasticsearch), 결제(KCP/토스페이먼츠), 푸시(FCM/APNs), 클라우드(AWS), AI(Claude/ChatGPT/Gemini).
  • 주문 1번에 8개 이상 시스템 협력.
  • 한 사람이 다 다루지 않음. 시작점이 중요.
