SECTION 01 · 도입

실무 IT 산업의 큰 그림

축하드립니다. 4주 동안 비전공자 입장에서 적지 않은 분량의 Python을 배웠습니다. 이제 Python을 손에 쥐고 실무 IT 산업의 큰 그림을 한번 둘러볼 차례입니다.

여러분이 매일 쓰는 인스타그램, 카카오톡, 배달의민족, 유튜브 같은 서비스는 Python 한 가지 언어로만 만들어지지 않습니다. 수십 개의 언어, 수백 개의 도구가 함께 움직여서 여러분이 화면에서 보는 "주문하기" 버튼 한 번이 작동합니다.

이 부록은 그림 한 장으로 시작합니다. 그 그림 한 장에 IT 산업의 핵심 구조가 다 들어 있어요. "이걸 다 알아야 하나?" 부담 가질 필요 없습니다. "이런 세계가 있구나" 정도만 보시면 됩니다.

SECTION 02 · 큰 그림

한 장으로 보는 IT 산업

인터넷 서비스는 보통 3개 영역이 협력해서 만들어집니다. Frontend(사용자가 보는 화면) ↔ Backend(서버에서 도는 처리) ↔ Database(데이터 보관 + 뒷단 서비스들). 아래 한 장에 모든 것이 들어 있어요.

그림 5.1 — IT 산업 한 장. Frontend(왼쪽) ↔ Backend(가운데) ↔ Database/뒷단(오른쪽)

이게 IT 산업의 큰 그림입니다. 여러분이 4주 동안 배운 Python은 가운데 Backend 영역에 자리 잡고 있어요. 이제 각 영역을 차례로 둘러봅시다 — 그림 위에 글로 살짝 덧붙이는 정도입니다.

SECTION 03 · Frontend

Frontend — 사용자가 보는 화면

Frontend는 사용자가 눈으로 보고 손으로 만지는 부분입니다. 인스타 앱을 켜면 보이는 피드, 좋아요 버튼, 댓글 입력창 — 이 모든 시각적 요소가 Frontend 코드로 만들어집니다.

어디서 도느냐에 따라 셋으로 나뉩니다:

  • Web — 크롬·사파리·엣지 같은 웹 브라우저에서 도는 코드. 네이버, 쿠팡, 구글 같은 사이트.
  • 모바일 앱 — 스마트폰에 설치하는 프로그램. 카톡, 인스타, 배달의민족.
  • 데스크톱 앱 — PC에 설치하는 프로그램. 카톡 PC 버전, Slack, VS Code.

Web 기초 3종 세트

Web Frontend를 배우려면 HTML, CSS, JavaScript 이 셋을 알아야 합니다. 이 셋은 한 묶음으로 다닙니다. HTML은 웹페이지의 뼈대(어떤 글, 어떤 이미지, 어떤 버튼이 있는지), CSS는 웹페이지의 옷(색깔·크기·위치 같은 모양), JavaScript는 웹페이지의 움직임(버튼 누르면 반응하는 등)을 정의합니다. 건물에 비유하면 골조 / 외관 / 전기·엘리베이터에 해당해요.

HTML/CSS/JS만으로 큰 사이트를 만들면 코드가 너무 복잡해집니다. 그래서 프레임워크(이미 만들어진 도구 모음)를 사용합니다. 가장 인기 있는 게 페이스북이 만든 React이고, 배우기 쉽다고 알려진 Vue, 구글의 Angular, 새로 부상하는 Svelte 같은 것들이 있어요.

모바일 앱 — iOS와 Android

모바일 앱은 두 운영체제로 나뉩니다. iOS(아이폰)는 애플의 Swift 언어를, Android(삼성·LG 등)는 구글이 미는 Kotlin 언어를 씁니다.

문제는 같은 앱을 두 번 만들어야 한다는 것. 그래서 한 번 짜면 둘 다 도는 크로스 플랫폼 도구들이 있어요 — 페이스북의 React Native(JavaScript)와 구글의 Flutter(Dart)가 대표입니다.

데스크톱 앱 — Electron

PC 프로그램은 요즘 Electron이라는 도구로 많이 만듭니다. 웹 기술(HTML+CSS+JS)로 데스크톱 앱을 만드는 방식이에요. VS Code, Slack, Discord, Figma 데스크톱이 모두 Electron으로 만들어졌습니다.

SECTION 04 · Backend

Backend — Python이 사는 곳

Backend는 사용자에게 보이지 않는 곳에서 일을 처리하는 부분입니다. 인스타에서 좋아요 버튼을 누르면, 그 정보가 인스타 회사의 서버 컴퓨터까지 가서 데이터베이스에 저장되고, 작성자에게 알림이 보내지고, 피드 정렬이 다시 계산됩니다. 이 모든 처리가 Backend의 일이에요.

서버 = 회사가 운영하는 컴퓨터

"서버(Server)"는 두 가지를 가리킵니다. 회사가 24시간 켜놓고 인터넷에 연결한 큰 컴퓨터를 가리키기도 하고, 그 컴퓨터에서 도는 백엔드 프로그램을 가리키기도 해요. 보통 둘 다를 묶어서 "서버"라 부릅니다.

Backend의 5가지 일

  1. 요청 받기 — Frontend가 보내는 API 요청을 받음 (4주차에서 배운 그 API)
  2. 인증 확인 — "이 요청 보낸 사용자가 진짜 본인인가?" 토큰 검증
  3. 비즈니스 로직 처리 — "이 사용자는 이 행동을 할 수 있는가?" 같은 규칙
  4. DB 입출력 — 데이터베이스에 저장하거나 꺼내옴
  5. 응답 반환 — 처리 결과를 JSON 형태로 돌려줌

Backend 언어들

Backend는 다양한 언어로 만들 수 있습니다. Python ★(여러분이 배운 그것!)은 Django/Flask/FastAPI 도구를 쓰며 데이터 분석·AI 연계가 강점입니다. Instagram·Netflix·토스 일부 시스템이 Python을 씁니다.

Java는 한국 기업에서 가장 많이 쓰는 백엔드 언어로, Spring Boot 프레임워크와 짝을 이룹니다. 네이버·카카오·토스·배달의민족 등 한국 대기업 대부분이 Java를 써요.

그 외에 JavaScript/TypeScript(Node.js — Frontend와 같은 언어로 백엔드를), Go(구글이 만든 언어, 동시 처리 성능 강함), Ruby(Ruby on Rails — GitHub·Airbnb이 사용), PHP(Facebook·Wikipedia가 시작한 언어), C#(마이크로소프트, .NET), Rust(안전성·성능 모두 뛰어난 새 언어) 등이 있어요.

Python으로 시작하는 백엔드 — 닿을 수 있는 거리

여러분이 4주 동안 Python을 배웠으니 다음 도구 중 하나를 익히면 백엔드 개발자로 시작할 수 있어요. 가장 작고 학습용으로 좋은 게 Flask, 현대적이고 인기 있는 게 FastAPI, 큰 사이트용은 Django입니다 (Instagram이 Django 사용).

마이크로서비스 — 큰 백엔드의 구조

배달의민족 같은 큰 서비스는 백엔드를 한 덩어리로 만들지 않고 작은 조각들로 나눕니다 — 이걸 마이크로서비스라고 해요. 회원 서비스, 가게 서비스, 주문 서비스, 결제 서비스, 배달 서비스, 알림 서비스 같이 독립적인 작은 백엔드 여러 개로 운영됩니다. 한 서비스가 잠깐 멈춰도 다른 서비스에 영향이 없어요.

실제로 우아한형제들 기술블로그(2016)에 따르면, 배달의민족이 마이크로서비스 + Java/Spring 조합으로 운영된다고 공개했습니다.

SECTION 05 · Database / 뒷단

Database / 뒷단 서비스 — 백엔드 뒤의 세계

백엔드는 혼자 일하지 않습니다. 백엔드 뒤에는 데이터를 보관하는 곳, 빨리 가져오기 위한 곳, 메시지를 전달하는 곳, 결제를 처리하는 회사, AI 서비스 등이 줄지어 있어요. 그림 5.1의 오른쪽 영역이 바로 이 부분입니다.

Database — 데이터를 보관하는 창고

데이터베이스(DB)는 데이터를 영구적으로 보관하는 창고입니다. 회원 정보, 게시글, 주문 내역, 댓글… 사용자가 입력한 모든 데이터가 결국 DB에 저장돼요.

가장 많이 쓰이는 방식이 관계형 DB(RDB)로, 데이터를 표(테이블) 형태로 저장합니다. 예를 들어 배달앱이라면 users 표, stores 표, orders 표 같은 표들을 만들고, 표끼리 서로 연결지어요. 은행·쇼핑몰·예약 시스템처럼 데이터가 정확하게 맞아야 하는 곳에 강합니다. 대표 제품이 MySQL(가장 흔함, 배달의민족도 사용), PostgreSQL(기능 풍부, 새 프로젝트에 인기), Oracle(유료, 대형 은행·금융권)입니다.

NoSQL은 표가 아닌 다른 방식으로 저장하는 DB들이에요. 유연한 구조가 강점입니다. MongoDB(JSON 형태 저장 — 상품마다 속성이 다른 쇼핑몰에 적합), Cassandra(여러 서버에 분산 저장 — 인스타·Netflix처럼 데이터 폭발하는 곳), DynamoDB(AWS의 관리형 DB) 등이 있습니다.

SQL — 백엔드의 다음 학습

관계형 DB에 접근하려면 SQL(Structured Query Language)이라는 언어를 씁니다. 예를 들어 "users 표에서 30살 이상 다 찾기"는 SELECT * FROM users WHERE age >= 30; 이렇게 써요. 모든 RDB 제품(MySQL·PostgreSQL·Oracle)이 SQL을 씁니다. 백엔드 개발자가 다음으로 배워야 할 가장 중요한 한 가지가 SQL입니다.

캐시 — 빨리 꺼내기

캐시(Cache)는 자주 쓰는 데이터를 미리 빠른 곳에 저장해두는 방법입니다. 인스타 피드를 새로고침할 때 매번 DB에서 처음부터 계산하면 너무 느려요. 그래서 한 번 계산한 피드를 메모리에 잠깐 저장해두면 다음에 0.01초 만에 꺼낼 수 있습니다 (디스크보다 100배 이상 빠름).

표준 도구가 Redis입니다. 캐시뿐만 아니라 로그인 세션 보관, 실시간 순위, 분산 락 등 다양하게 쓰여서 거의 모든 백엔드에 Redis 하나는 있다고 봐도 됩니다. Memcached는 옛날부터 쓰인 단순한 캐시예요.

메시지 큐 — 시스템 간 우편함

메시지 큐는 한 시스템에서 다른 시스템으로 메시지를 보낼 때 줄 세워주는 우편함 같은 도구입니다.

왜 메시지 큐가 필요한가?

배민에 주문이 들어오면 백엔드가 매장·라이더·푸시·분석 시스템 4곳에 알려야 합니다. 만약 직접 차례로 호출하면 한 곳이라도 멈추면 다 멈추고 응답이 늦어져요. 대신 메시지 큐에 "주문 들어왔어!" 한 번만 던지면 4곳이 각자 알아서 가져갑니다. 한쪽이 멈춰도 다른 쪽 영향 없어요.

표준 도구가 Kafka입니다. 큰 규모 처리에 강하고, LinkedIn·Netflix·Uber·배달의민족이 모두 사용해요. 실제로 우아한형제들 기술블로그(2025)에 따르면 배달의민족이 Kafka로 주문 이벤트를 처리한다고 공개했습니다. RabbitMQ는 작은 규모에 적합한 가벼운 메시지 큐입니다.

검색 엔진 — 찾기를 빠르게

DB에도 검색 기능이 있지만 텍스트 검색(예: "치킨"이 들어간 가게 다 찾기)은 느립니다. 그래서 전용 검색 엔진을 따로 둬요. 오타 자동 보정("치킹" → "치킨"), 비슷한 단어 같이 보여주기, 인기·평점 순 정렬을 0.1초 안에 처리합니다.

표준 도구가 Elasticsearch로, 쿠팡 검색·GitHub 검색·배민 가게 검색 등에 쓰입니다. OpenSearch는 AWS가 만든 무료 버전이에요.

결제 — 한국의 PG사

결제 대행사 (PG사, Payment Gateway)는 온라인 결제를 대신 처리해주는 전문 회사입니다. 한국법상 쇼핑몰이 사용자 카드 정보를 직접 보유하면 불법이라, 전문 회사에게 맡기는 구조예요. 쇼핑몰 → PG사 → 카드사 → 은행 → 다시 쇼핑몰 흐름으로 결제가 진행됩니다.

한국에는 NHN KCP, KG이니시스, 토스페이먼츠, 나이스페이먼츠 같은 회사들이 있어요. 그 외에 카카오페이·네이버페이·페이코 같은 간편결제도 있습니다.

푸시 알림 — 모바일 앱의 표준

푸시 알림은 앱이 닫혀있어도 화면 위에 뜨는 메시지예요. 카톡 알림, 인스타 좋아요 알림 같은 거. 푸시는 직접 보낼 수 없고 반드시 운영체제 회사를 거쳐야 합니다. 안드로이드는 구글의 FCM(Firebase Cloud Messaging), iOS는 애플의 APNs(Apple Push Notification service)이 표준이에요. 둘 다 무료입니다.

클라우드 — 컴퓨터 빌려쓰기

옛날에는 회사가 서버 컴퓨터를 직접 사서 회사 건물에 두고 운영했지만, 요즘은 클라우드에서 빌려씁니다. "내가 직접 컴퓨터 안 사고 인터넷 너머의 회사가 운영하는 컴퓨터를 빌려쓰는" 방식이에요. 사용한 만큼만 비용을 냅니다.

빅3가 아마존의 AWS(가장 오래되고 서비스 많음, 한국 IT 기업 대부분 사용), 마이크로소프트의 Azure(기업 시장 강함), 구글의 GCP(AI·머신러닝 강함)입니다. 한국에는 네이버 클라우드, NHN Cloud도 있어요. 공공기관·금융권이 선호합니다.

컨테이너 — 어디서든 똑같이 돌리기

"내 컴에선 잘 돌았는데요"

개발자 노트북에선 코드가 잘 돌았는데 회사 서버에 올리면 안 도는 경우가 많았어요. Python 버전·라이브러리·환경 변수가 달라서요. Docker는 코드+필요한 환경을 통째로 박스(컨테이너)로 묶어서, 어떤 컴퓨터에 가져가도 똑같이 돌게 해줍니다. "내 컴에선 잘 돌았는데" 문제가 사라졌어요.

Kubernetes(쿠버네티스, k8s)는 컨테이너 수십·수백 개를 자동으로 관리하는 도구입니다. 한 컨테이너가 죽으면 자동 재시작, 트래픽 몰리면 자동 확장. 큰 규모 서비스의 표준이에요.

★ AI 서비스 — IT 산업의 새 카테고리

1주차 첫 시간에 약속드린 부분 — "AI 시대에 코딩 알면 좋다"고 했죠. 이제 그 약속을 회수합니다. IT 산업에 새로 추가된 큰 카테고리가 AI 서비스예요. 백엔드 개발자가 외부 AI를 호출해서 자기 서비스에 똑똑함을 더하는 게 새로운 표준이 됐습니다.

대표 회사가 Anthropic(Claude — 코드 생성에 강함, 기업 시장 1위), OpenAI(ChatGPT — 일반인 인지도 1위), Google(Gemini — 멀티모달 강함), Meta(Llama — 오픈소스, 무료 다운로드) 등입니다.

여러분이 4주차에 배운 Python으로 OpenAI나 Claude API를 호출하는 게 지금 가장 인기 있는 신입 개발자 프로젝트 중 하나예요. 4주차 pyfetch나 데스크톱 Python의 requests로 API 호출하는 방식이 그대로 적용됩니다.

SECTION 06 · 시뮬레이터

시뮬레이터 — 배달의민족 주문 흐름

위에서 본 모든 것이 어떻게 함께 움직이는지 시뮬레이터로 따라가 봅시다. 배달의민족에서 주문 한 번을 누를 때 일어나는 일을 10단계로 보여드립니다.

✓ 표시는 우아한형제들이 공개한 사실(출처 명시), ❓ 표시는 공개되지 않아 일반 IT 패턴으로 추측한 부분입니다.

주문 한 번에 최소 8개 시스템이 협력합니다. 배민 한 회사 안의 시스템들만 셈한 것이고, 클라우드(AWS)·결제 카드사·푸시 시스템까지 더하면 더 많아요. 인스타 사진 업로드, 카톡 메시지, 유튜브 영상 — 다른 서비스도 비슷한 다단계 흐름 위에서 돕니다.

"내가 이걸 다 알아야 하나?" 부담 X. 한 사람이 다 다루지 않습니다. 백엔드·프론트엔드·DevOps·데이터·모바일 개발자가 각자 한두 영역에 집중하고 협력해서 큰 서비스를 만들어요. 여러분이 Python으로 시작했다면 Backend 또는 뒷단 일부에 자연스럽게 닿게 됩니다.

SECTION 07 · 마무리

여기까지 — 그리고 앞으로

부록까지 따라오시느라 수고 많으셨습니다. 이제 여러분은 단순히 Python 문법을 아는 사람이 아니라, "인터넷 서비스가 어떻게 돌아가는지 큰 그림을 본 사람"입니다.

1주차의 약속 회수 — AI 시대

1주차 첫 시간에 약속드린 게 있어요. "AI 시대에는 'AI에게 일 시키는 사람'과 'AI가 일 시키는 사람'이 나뉠 것"이라고요. 이제 그 의미가 더 분명해졌을 거예요.

AI가 코드를 빠르게 만들어주는 시대지만, 그 코드가 어디에 들어가는지 알아야 검증할 수 있습니다. "백엔드", "DB", "API", "인증" 같은 단어를 들으면 어느 위치인지 안다는 게 큰 차이를 만듭니다. AI가 "Redis 써서 캐시 처리하시면 됩니다"라고 답해도, 여러분은 "왜 캐시가 필요한지" 알 수 있죠. 새로 부상하는 AI 서비스(Claude, ChatGPT, Gemini) 자체도 결국 백엔드가 호출하는 뒷단 서비스입니다.

다음 학습 — Python 깊이 + 백엔드

이 강의는 4주짜리 기초였어요. 다음 단계로 갈 수 있는 길:

  • Python 더 깊이 — 클래스, 데코레이터, 람다, 컴프리헨션, 비동기. 모듈/패키지.
  • 백엔드 시작 — Flask 또는 FastAPI로 작은 API 서버 만들기. 4주차 API 호출의 반대편.
  • SQL과 데이터베이스 — PostgreSQL 또는 MySQL 설치하고 직접 표 만들고 쿼리 날려보기.
  • Frontend — HTML/CSS/JavaScript 기초 → React. 직접 화면 만들어보기.
  • AI 활용 — OpenAI 또는 Claude API를 Python으로 호출하는 작은 프로젝트.
  • 클라우드 — AWS 무료 계정으로 작은 서버 띄우고 Python 코드 돌려보기.

마지막으로

실무 IT 산업에는 끝없이 새로운 도구가 나옵니다. 5년 후에는 지금 본 도구들 중 일부가 사라지고 새 도구가 자리 잡을 거예요. 그게 IT의 본질입니다.

하지만 Frontend ↔ Backend ↔ Database라는 큰 그림 자체는 변하지 않을 겁니다. 이 흐름은 인터넷의 본질에 가까우니까요. 이름만 바뀔 뿐 같은 자리에 같은 종류의 도구가 자리잡습니다.

여러분이 4주 동안 배운 변수·함수·반복·조건·자료구조도 마찬가지입니다. 어떤 언어를 써도, 어떤 시대가 와도, 이 기본 개념은 변하지 않아요.

5주차 핵심 정리
  • 인터넷 서비스의 큰 그림: Frontend ↔ Backend ↔ Database/뒷단.
  • Frontend: HTML/CSS/JS, React, Vue, Swift, Kotlin, Flutter.
  • Backend: Python, Java, JavaScript, Go, Ruby, PHP, C#, Rust. 큰 백엔드는 마이크로서비스로.
  • Database/뒷단: DB(MySQL/PostgreSQL/MongoDB), 캐시(Redis), 메시지 큐(Kafka), 검색(Elasticsearch), 결제(KCP/토스페이먼츠), 푸시(FCM/APNs), 클라우드(AWS/Azure/GCP), AI(Claude/ChatGPT/Gemini).
  • 주문 1번에 8개 이상의 시스템이 협력합니다.
  • "이걸 다 알아야 하나" 부담 X — 한 사람이 다 다루지 않음. 시작점이 중요.
